# Business Ontology Skill

Русскоязычный Agent Skill для разработки бизнес-онтологии: модели бизнес-реальности компании, бизнес-замысла, модуля, производственной системы, интерфейса, состояний, решений и дрифта.

Скилл помогает агенту вести онтологическую сессию в стиле `grill-with-docs`: задавать один сильный вопрос за раз, предлагать рекомендуемый ответ, проверять противоречия, не подхалимничать и фиксировать принятые решения в файлах.

## Для Чего

- Первая онтологическая сессия по компании, модулю или бизнес-замыслу.
- Продолжение уже существующей онтологии без переписывания с нуля.
- Аудит дрифта: что изменилось, устарело или конфликтует.
- Упаковка понятий, модулей, производственных систем, интерфейсов, процессов, метрик и правил.
- Подготовка агент-читаемой модели бизнес-реальности.

`SKILL.md` содержит только ядро поведения. Подробная структура, шаблоны карточек и проверки вынесены в `references/`, чтобы агент загружал их только при необходимости.

## Ключевые Принципы

- Бизнес-онтология - это модель бизнес-реальности, не RDF/OWL/SHACL.
- Один фокусный вопрос за раз.
- Рекомендуемый ответ, если контекст уже дает разумное направление.
- Проверка неопределенности, конфликтов, дрифта и источников.
- Запрет на подхалимство: конфликтующие правки нужно разбирать, а не принимать автоматически.
- Ленивое создание файлов: структура появляется только там, где есть материал.
- Durable state: решения, диффы и открытые вопросы должны жить в артефактах, а не только в чате.

## Установка

Скопируйте папку скилла в директорию навыков вашего агента.

```bash
cp -R business-ontology ~/.codex/skills/business-ontology
```

Если репозиторий клонируется напрямую как папка скилла:

```bash
git clone https://github.com/Vladick-Pick/business-ontology.git ~/.codex/skills/business-ontology
```

## Проверка

```bash
python ~/.codex/skills/.system/skill-creator/scripts/quick_validate.py .
```

Для ручной проверки поведения используйте `evals/activation-and-behavior.md` и `references/pressure-tests.md`.

## Статус

Скилл готов к первой публичной апробации. Он намеренно остается markdown-only: без исполняемых скриптов, внешних зависимостей и скрытых действий.

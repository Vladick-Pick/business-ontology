## Findings considered and rejected / deferred

Отложено сознательно (не потеряно — кандидаты на 0.9.0):

- **Graph impact API** (`query_registry.py`: neighbors/impact BFS + ресурс
  `model/impact/{id}`) — высокая ценность, но не блокирует релиз; после 002.
- **Time-travel** (`build_registry.py --at <git-rev>`) и **diff_registry.py** —
  после суперсессии (004) и impact API.
- **AUDIT.md + scripts/audit_model.py** (исполняемый независимый аудит) — после 002
  (аудит должен проверять v2-контракт, писать дважды не стоит).
- **Матрица автономии ревью risk×evidenceGrade + TTL пакетов + метрика
  пропускной способности** — требует решения владельца о границах автономии
  (человеческое решение, не техническое).
- **Независимый грейдер evidenceGrade/sourceRisk, двухпроходная экстракция,
  регрессии из отклонённых пакетов, drift-detection бенчмарк, калибровка
  confidence** — следующий слой после 006.
- **Позиционный линтер текста чата** (`position_lint.py`) — после 003; требует
  накопленных реальных трасс.
- **Archify-экспортёр** (`scripts/export_diagrams.py`: карточка → archify-JSON →
  HTML/PNG со штампом ontologyRevision) — канал публикации; после 001 и 002.
- **elkjs вместо dagre во вьюере, кликабельность узлов, zoom/pan** — качество
  раскладки; после 001.
- **Diagram-spec схема + скилл для агента** — целиться в схемы archify (v3 JSON IR
  у них в роадмапе); вместе с archify-экспортёром.
- **Ужесточение transitional warnings → errors** (SOFT_REQUIRED_ATTRS,
  links_validate.py:250): механизм — failing-тест, привязанный к версии пакета
  (при version ≥ 0.10.0 warning-набор обязан быть пуст) — чтобы «одна переходная
  версия» не зависла навсегда. Кандидат в ближайший релизный план.
- **Contested/stale-оверлей для читателей** (находка Codex к «нет срочной полосы»):
  read-only флаг в проекциях interpret/registry — «accepted, но есть неразобранный
  конфликтующий пакет от <даты>». Не меняет истину, меняет честность ответа.
  Кандидат на 0.9.0.
- **trust-by-claim в source map** (доверие источнику по роду факта — находка
  стресс-теста Д4) — маленькая правка, но трогает валидатор и review-ux; вместе
  с 002-followup.
- Отклонено: **отдельный тип «бизнес-процесс»** — разводится на process /
  production-system+lifecycle / цепочку интерфейсов (см. docs/specs/…data-model-v2.md §1);
  **goal/event/risk/оргиерархия как типы** — watch-list с критериями введения
  (спека §9.3), вводить только решением.

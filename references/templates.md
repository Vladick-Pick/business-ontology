# Шаблоны Карточек

Используй эти шаблоны только когда реально создаешь или обновляешь соответствующую карточку.

## Правило Заполнения

Не оставляй пустые поля и пустые разделы.

Если информации нет, используй одно из значений ниже.

- `неизвестно` - важно, но пока не определено;
- `не применимо` - раздел не нужен для этой сущности;
- `кандидат` - вероятное определение, но еще не утверждено;
- `гипотеза` - предположение без достаточного источника.

Все важное `неизвестно`, `конфликт` и `гипотеза` также дублируй в `08-дрифт-и-открытые-вопросы.md` или в разделе `Дрифт И Открытые Вопросы` карточки.

## Фронтматтер И Связи

У каждой карточки сверху - YAML-фронтматтер со стабильным `id` (не меняется при переименовании) и блоком `связи` с типами из закрытого короткого списка (см. [ai-ready.md](ai-ready.md)). `id` + `связи` делают модель запрашиваемой агентом и склеиваемой по `id` (например финансовый оверлей цепляется к карточке по её `id`).

Правила: типы связей бери только из `ai-ready.md`; если связи еще нет - не пиши пустой блок; каждый `id` в `связи` должен резолвиться в существующую карточку. Раздел `## Связи` в теле карточки (если есть) - необязательная человеческая проза; машинная истина о связях живет во фронтматтере.

## Как Есть И Как Должно (опционально)

Карточка по умолчанию описывает, как реально работает сейчас. Если регламент («как должно») расходится с практикой («как есть») и разрыв важен для решения - добавь в карточку опциональный блок и продублируй его в `08-дрифт-и-открытые-вопросы.md` с типом `зазор`:

```
зазор:
  как-есть: <что реально происходит>       (источник: наблюдение/кейс)
  как-должно: <что предписано регламентом>  (источник: регламент/цель)
  почему-важно: <какое решение задевает>
```

Не добавляй этот блок, если расхождения нет - помечается только реальный зазор.

## Карточка Интерфейса

